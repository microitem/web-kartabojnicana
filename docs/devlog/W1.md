# W1 — Devlog (implementer)

> Implementer-owned exception (`docs/devlog/` → implementer, viď `AGENTS.md` §6).
> Základ prezentačnej stránky `kartabojnicana.sk` podľa `docs/01-zadanie-implementacia.md`.

## Stack

Next.js 16 (App Router) + TypeScript + Tailwind v4 — zhodné s appkou
(`microitem/kartabojnicana`), žiadna databáza, žiadne tajomstvá okrem
`NEXT_PUBLIC_APP_URL` (nie citlivé, len konfigurácia odkazu na appku).

## Čo je hotové

- Sekcie podľa `docs/00-zamer.md` §3 a `docs/01-zadanie-implementacia.md`: Hero,
  Čo je Karta Bojničana, Pre koho (Rezident/Turista), Ako to funguje (3 kroky),
  Pre prevádzky, Katalóg partnerov (placeholder "čoskoro"), FAQ, pätička.
- **Značka Veža** (R-2026-34): zdieľaný `Logo` komponent + `public/brand/veza-mark*.svg`
  — identický mask/gradient markup ako v appke, aby obe domény zobrazovali
  rovnaký znak. Dobré Bojnice je len prevádzkovateľ, v pätke ("Prevádzkuje
  platforma Dobré Bojnice"), nie hlavné logo (R-2026-31).
- **Obsah (W2)** — texty zo schváleného dizajnového demo (`docs/design/kartabojnicana-web.demo.html`)
  prevzaté ako ilustračný návrh, **nie sú CEO schválené**. Persistentný banner
  "NÁHĽAD" na stránke to jasne označuje. Konkrétne vymyslené čísla/štatistiky
  z demo (napr. "≤10 s", "100 %" v sekcii pre prevádzky) nahradené `[N]`
  placeholdermi s vysvetľujúcou poznámkou — nie sú prezentované ako reálne dáta.
  FAQ odpovede sú faktické (zodpovedajú skutočnému správaniu appky — OTP overenie,
  D1 záznam zľavy, D4 rodné číslo, EÚ hosting), nie marketingové tvrdenia.
- **Odkazy do appky (W5)** — centralizované v `src/lib/appLinks.ts`
  (`NEXT_PUBLIC_APP_URL`, default `https://app.kartabojnicana.sk`), všetky
  overené že smerujú na reálne existujúce cesty appky (`/register`,
  `/register-resident`, `/merchant/login`). Odkaz na "Ochrana osobných údajov"
  vedome vynechaný — appka takú stránku ešte nemá (bola by to nefunkčná
  kotva/W5 porušenie).
- **Prístupnosť (W3)** — natívny `<details>/<summary>` pre FAQ (klávesnicovo
  ovládateľné bez JS), `:focus-visible` outline, sémantické nadpisy, `aria-label`
  na dekoratívnej karte v hero sekcii. Responzívne overené (desktop + mobil,
  vizuálne cez Playwright).
- **Infra** — `Dockerfile` (dev/builder/production stage, zhodný pattern ako
  appka), `docker-compose.yml` (port `3037`, zaregistrovaný v
  `.github/PORT-CONFLICTS.md`), `.dockerignore`, `.env.example`. CI
  (`.github/workflows/ci.yml`): lint + typecheck + build.

## Dodatok (na žiadosť CEO, pred schválením na DEV)

CEO počas prehliadky DEV (`:3037`) požiadal o "jemné moderné efekty na celú
stránku". Doplnené čisto vizuálne/CSS, bez zmeny obsahu či štruktúry:

- Dva jemné rozmazané gradientové "bloby" (oranžová/fialová, nízka priehľadnosť)
  pomaly plávajúce za hero sekciou; jemné plávanie dekoratívnej hero karty.
- Hover-lift + tieň na kartách (výhody, rezident/turista, náhľad prevádzky) a
  plynulé prechody na všetkých tlačidlách/odkazoch.
- Rotujúca šípka na FAQ položkách (`group-open:rotate-180`), vlastný chevron
  namiesto natívneho markera.
- Scroll-reveal (fade-in-up) cez `animation-timeline: view()` — iba v
  prehliadačoch s podporou (`@supports`), inak obsah ostáva vždy plne viditeľný.
- Celý blok efektov je v `@media (prefers-reduced-motion: no-preference)` —
  pri preferencii obmedzeného pohybu sa nič neanimuje (W3/WCAG 2.1 AA).

Overené: `npm run lint` + `npm run typecheck` čisté, kontajner force-recreate
(anonymný `.next` volume), vizuálne cez Playwright na desktop (1440×900) aj
mobile (390×844) — žiadny horizontálny overflow, hover/FAQ interakcie funkčné.

**Dodatok 2** (CEO feedback po prvej vizuálnej kontrole): hero "bloby" boli bez
`blur` a s veľkým záporným offsetom, takže sa na okrajoch orezávali ako plochý
tvar, nie mäkký kruh. Opravené — menšia veľkosť, kladný offset (plne vnútri
`overflow-hidden` kontajnera), `blur-3xl` pre mäkký žiar. Zároveň doplnené
efekty aj na texty naprieč celou stránkou: scroll-reveal (fade-in-up cez
`animation-timeline: view()`, `@supports`-gated) na nadpisoch/eyebrow
labeloch/odsekoch vo všetkých sekciách, animovaný podčiark pri hoveri
(`link-underline`, CSS `background-size` prechod) na navigácii v hlavičke aj
v pätičke. Overené Playwright — bloby celé (bez orezania) na desktop aj
mobile, žiadny horizontálny overflow, podčiarky bez posunu layoutu, žiadny
text natrvalo neviditeľný (reveal je čisto scroll-pozičný).

**Dodatok 3** (CEO feedback): hero pozadie (bloby) bolo orezané na šírku
`max-w-6xl` obsahového stĺpca namiesto celej šírky obrazovky — na širokých
monitoroch (>1152px) vyzeralo pozadie ako orámované do užšieho boxu.
Opravené — vonkajšia `<section>` je teraz plnou šírkou (`relative
overflow-hidden`, bez `max-w`), obsah (text + karta) je vnorený `<div
className="mx-auto max-w-6xl">`, bloby sú pozicované percentuálne (`left-[6%]`
/ `right-[8%]`) voči celej šírke. Overené Playwright pri 1920×1080 (`section`
šírka = `window.innerWidth`), 1440×900 aj mobile — bez horizontálneho
overflow.

## Príprava produkčného nasadenia (po CEO schválení DEV, `:3037`)

CEO schválil obsah/vizuál na DEV (§4 krok 3 zadania) → pripravené (nie
spustené) kroky pred krokom 4 ("Produkčné nasadenie = CEO gate"):

- `docker-compose.production.yml` (nový) — mirror `kartabojnicana` app repa:
  pull hotového GHCR image, žiadny publikovaný host port (reverse-proxy cez
  zdieľaný `nginx-proxy-manager`), žiadna DB/secrets — len
  `NEXT_PUBLIC_APP_URL` s fallbackom.
- `.github/workflows/deploy.yml` (nový) — tag-triggered (`v*.*.*`), rovnaký
  digest-check + backup/rollback vzor ako appka, zjednodušené (bez migrácií).
- GitHub Secrets na tomto repe nastavené (`VPS1_HOST/PORT/USER/SSH_KEY`
  zdieľané s appkou, `VPS1_PROJECT_DIR=/opt/projects/web-kartabojnicana` nové).
- `/opt/projects/web-kartabojnicana` na VPS1 pripravený (`git clone`) —
  inertné, žiadny kontajner tam ešte nebeží, NPM naň nič nesmeruje.

**Nasadené** (CEO GO, R-2026-35 v `kartabojnicana/rozhodnutia.md`): `v0.1.0` prvý
pokus zlyhal — VPS1 docker default address pool (`192.168.0.0/16` rozdelený na
/20) bol plne vyčerpaný (31 existujúcich sietí na hostiteľovi); opravené
explicitným `172.31.0.0/16` (posledný nevyužitý `/16` v `172.17-172.31`
rozsahu, overené `docker network inspect` naprieč všetkými existujúcimi
sieťami) v `docker-compose.production.yml`. `v0.1.1` nasadené úspešne —
kontajner `web-kartabojnicana-app-prod` beží zdravo na VPS1, smoke test
(`wget` priamo do kontajnera) vracia stránku. **Zatiaľ bez verejného
smerovania** — NPM naň ešte nesmeruje (viď nižšie).

**Ostré spustenie dokončené:** CEO prepol NPM proxy host `kartabojnicana.sk`/
`www` na `web-kartabojnicana-app:3000` sám (Access List/Proxy Hosts nie sú
moja doména — implementer needituje zdieľanú NPM konfiguráciu priamo). Chýbalo
pripojenie `nginx-proxy-manager` kontajnera na sieť `web_kartabojnicana_prod_net`
(502 pri prvom teste) — doplnené (`docker network connect`, aditívne, bez
reštartu NPM, rovnaký vzor ako appka má s `kartabojnicana_prod_net`). Overené
naostro: `https://kartabojnicana.sk` aj `www.` → 200. Dočasný redirect v appke
(`src/middleware.ts`) odstránený tam (mŕtvy kód, appka root-host requesty už
nedostáva).

**Dodatok 4 (CEO, 2026-07-22 večer):** kým neprebehne verejné oznámenie/
spustenie, CEO chce web dočasne prístupný len interne. Riešenie: **NPM Access
List** (`kartabojnicana-internal`, HTTP Basic Auth, bez IP obmedzenia — funguje
odkiaľkoľvek vrátane reviewer cloud agenta) pripojený k proxy hostu
`kartabojnicana.sk`/`www`; CEO to nastavil sám v NPM UI (rovnaký dôvod —
zdieľaná infra). Overené `curl`: root/`www` → `401`, `app.kartabojnicana.sk`
nezmenené (`200`). Zapísané R-2026-36 v `kartabojnicana/rozhodnutia.md`.

## Mimo rozsahu tejto iterácie

  druhá iterácia podľa zadania, nerobené teraz.
- Produkčné nasadenie na `kartabojnicana.sk` (root) — CEO gate, čaká na
  schválenie obsahu na DEV. Poznámka pre koordináciu: appka
  (`microitem/kartabojnicana`) má momentálne dočasný redirect z root domény na
  `app.*` (Issue #1 [33]) — pri produkčnom nasadení tohto webu bude treba tento
  redirect v appke odstrániť/upraviť tak, aby root doména smerovala sem.
